# Demo Data Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let visitors load sample ships, gear, and engineering stats to explore the app without their own game export file.

**Architecture:** A localStorage flag (`demo_data_loaded`) tracks demo mode. Static demo data is written directly to the existing storage keys. A sidebar badge shows demo state with a clear action. The AuthProvider guards against syncing demo data to Supabase on sign-up.

**Tech Stack:** React 18, TypeScript, localStorage, existing UI components (Button, ConfirmModal)

**Spec:** `docs/superpowers/specs/2026-04-23-demo-data-design.md`

### Task 2: Create demo data utilities

**Files:**
- Create: `src/utils/demoData.ts`

This file contains the static demo data (5 ships, ~20 gear pieces, engineering stats for 2 ship types) and three helper functions.

- [ ] **Step 1: Create `src/utils/demoData.ts`**

The file exports:
- `DEMO_SHIPS: Ship[]` â€” 5 ships with varied factions, rarities, roles, levels. 3 ships have equipment references pointing to gear IDs. 2-3 ships have refits.
- `DEMO_INVENTORY: GearPiece[]` â€” ~18-20 gear pieces across all 6 slots for the 3 equipped ships. Mix of sets (ATTACK, DEFENSE, SPEED, CRITICAL, FORTITUDE), rarities (uncommon-legendary), levels (4-16), stars (1-5), 1-4 substats.
- `DEMO_ENGINEERING_STATS: EngineeringStats` â€” Stats for ATTACKER and DEFENDER ship types with flat stats (hp, attack, defence) and one percentage stat (crit).
- `isDemoDataLoaded(): boolean` â€” Reads `localStorage.getItem(StorageKey.DEMO_DATA_LOADED) === 'true'`.
- `loadDemoData(): void` â€” Writes `DEMO_SHIPS` to `StorageKey.SHIPS`, `DEMO_INVENTORY` to `StorageKey.INVENTORY`, `DEMO_ENGINEERING_STATS` to `StorageKey.ENGINEERING_STATS`, sets `StorageKey.DEMO_DATA_LOADED` to `'true'`, then calls `window.location.reload()`.
- `clearDemoData(): void` â€” Removes `StorageKey.SHIPS`, `StorageKey.INVENTORY`, `StorageKey.ENGINEERING_STATS`, and `StorageKey.DEMO_DATA_LOADED` from localStorage, then calls `window.location.reload()`.

**Important data relationships:**
- Each gear piece's `id` must be a valid UUID (use `crypto.randomUUID()` style, but hardcode them as static strings).
- Each equipped ship's `equipment` object maps slot names to gear piece IDs: `{ weapon: 'gear-uuid-1', hull: 'gear-uuid-2', ... }`.
- Each gear piece assigned to a ship must have `shipId` set to that ship's ID.
- Ship IDs are also hardcoded UUIDs.
- Refit IDs are hardcoded UUIDs. Each refit has 1-2 stats.

**Ship data guidelines:**
- Use real-sounding ship names (can be made up, doesn't need to match `ship_templates`).
- Base stats should be proportional to level/rarity (higher level + rarity = higher stats).
- All ships need `baseStats` with all fields: `hp`, `attack`, `defence`, `hacking`, `security`, `crit`, `critDamage`, `speed`, `healModifier`, `hpRegen`, `shield`, `defensePenetration`, `shieldPenetration`.

**Gear data guidelines:**
- `mainStat` is always present with `name`, `value`, `type`.
- `subStats` array has 1-4 entries, each with `name`, `value`, `type`.
- `setBonus` is a string from the gear set constants (e.g., `'ATTACK'`, `'DEFENSE'`).
- `slot` is one of: `'weapon'`, `'hull'`, `'generator'`, `'sensor'`, `'software'`, `'thrusters'`.
- `rarity` is one of: `'common'`, `'uncommon'`, `'rare'`, `'epic'`, `'legendary'`.

### Task 3: Create DemoDataSection component

**Files:**
- Create: `src/components/home/DemoDataSection.tsx`

- [ ] **Step 1: Create the component**

`DemoDataSection` is a simple component that:
1. Checks if data already exists in localStorage (`StorageKey.SHIPS` or `StorageKey.INVENTORY`).
2. If data exists and user clicks "Load Demo Data", shows a `ConfirmModal` warning that existing data will be replaced.
3. If no data exists, calls `loadDemoData()` directly.
4. Uses `Button` with `variant="secondary"` and `FlaskConical` icon from `lucide-react`.

```tsx
import React, { useState } from 'react';
import { FlaskConical } from 'lucide-react';
import { Button, ConfirmModal } from '../ui';
import { StorageKey } from '../../constants/storage';
import { loadDemoData, isDemoDataLoaded } from '../../utils/demoData';

export const DemoDataSection: React.FC = () => {
    const [showConfirm, setShowConfirm] = useState(false);

    const hasExistingData = () => {
        return !!localStorage.getItem(StorageKey.SHIPS) || !!localStorage.getItem(StorageKey.INVENTORY);
    };

    const handleLoadDemo = () => {
        if (hasExistingData()) {
            setShowConfirm(true);
        } else {
            loadDemoData();
        }
    };

    if (isDemoDataLoaded()) return null;

    return (
        <>
            <div className="border-t border-dark-border pt-4 mt-4 text-center">
                <p className="text-theme-text-secondary text-sm mb-3">
                    No game data? Try the app with sample data.
                </p>
                <Button variant="secondary" onClick={handleLoadDemo} className="gap-2">
                    <FlaskConical size={16} />
                    Load Demo Data
                </Button>
            </div>

            <ConfirmModal
                isOpen={showConfirm}
                onClose={() => setShowConfirm(false)}
                onConfirm={() => {
                    setShowConfirm(false);
                    loadDemoData();
                }}
                title="Load Demo Data"
                message="You already have data. Loading demo data will replace your current ships, gear, and engineering stats. Continue?"
                confirmLabel="Replace with Demo Data"
                cancelLabel="Cancel"
            />
        </>
    );
};
```

In `src/contexts/AuthProvider.tsx`:


2. In the `migrateDataForNewUser` function (starting at line 29), add a guard at the very beginning of the `try` block (after line 30), before the migration start event dispatch:

```typescript
// If demo data is loaded, clear it instead of syncing to Supabase
if (localStorage.getItem(StorageKey.DEMO_DATA_LOADED) === 'true') {
    localStorage.removeItem(StorageKey.SHIPS);
    localStorage.removeItem(StorageKey.INVENTORY);
    localStorage.removeItem(StorageKey.ENGINEERING_STATS);
    localStorage.removeItem(StorageKey.DEMO_DATA_LOADED);
    window.dispatchEvent(new Event('app:migration:end'));
    return;
}
```

This ensures that when an anonymous user with demo data signs up, the demo data is cleared and nothing is synced to Supabase. They start fresh.
